Phase 3  ·  EPIC-01 Backend API  ·  Apr 6–7
#1
feat(backend): EPIC-01 Inventory Management API
merged backend Apr 6 → Apr 7 epic-01-backend → master
▶
◆ Key Decisions Made
◆ Two separate endpoints for product creation — POST /api/products (catalog) + POST /api/stores/{id}/inventory (store linkage) Kept catalog and store concerns separate; simpler mental model at this stage
◆ Soft delete via inventory.is_active = false (per-store only) Products remain in catalog; only the store–product link is deactivated
◆ CurrentUserService stub for store ID derivation Swappable later without touching business logic
◆ Automatic transaction recording via domain events (@TransactionalEventListener) Audit trail without coupling service layer to recording logic
◆ Flat ProductInventoryResponse DTO Frontend needs one shape; avoids nested object traversal on the client

Phase 7  ·  Contract Consistency & DRY Cleanup  ·  Apr 14 (in review)
#4
refactor: backend contract consistency, DRY cleanup, and frontend UI normalization
open backend frontend refactor Apr 14 → present feat/epic-01-complete → master
▶
◆ Key Decisions Made
◆ Introduce StoreNotFoundException, SupplierNotFoundException → 404 All missing-resource cases should behave consistently; previous gap caused silent 500s
◆ Strict quantity semantics: receive/sell require > 0; adjust requires ≠ 0 and always records ADJUSTMENT Removed Math.abs() normalization — callers must pass correct sign, preventing silent data corruption
◆ Move DTO conversion from controller into mapper layer Controllers become pure orchestration; mappers own shape transformation
◆ Shared static ProductRequest.validateCore(…) used by both create and update DTOs Cross-field validation logic was duplicated — single source of truth
◆ Move store lookup-or-throw into StoreService; delete SaleRequest, unused repo methods Controllers should orchestrate, not own domain queries; dead code removed to reduce surface area
◆ Hide Dismiss / Update Threshold buttons behind isManager guard Manager-only actions leaked into associate view — guarded by role check
◆ Upgrade backend to JDK 21 (pom.xml + Dockerfile) Align with LTS; unlocks virtual threads and pattern matching for future use
◆ Normalize rounded-none → rounded-md; align edit page to shared design tokens UI inconsistencies accumulated across pages; single design language pass to fix them all
